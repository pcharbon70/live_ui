# `.spec`

This folder is the canonical `spec_led_ex` workspace for `live_ui`.

## Layout

- `README.md`: workspace guidance
- `specs/*.spec.md`: authored subject specs
- `state.json`: derived index and verification state generated by `mix spec.plan`, `mix spec.verify`, and `mix spec.check`

## Commands

- `mix spec.check`: canonical Spec Led plan + verify flow provided by `spec_led_ex`
- `mix live_ui.spec.check --strict`: repo-local governance/compliance overlay for `live_ui`

## Notes

- Authored specs remain in fenced JSON blocks even though `spec_led_ex` is YAML-first. JSON is accepted by the parser.
- Repo-local `spec-governance` blocks are preserved as local metadata for the `live_ui` governance overlay.
